# 提案機会検知くん リアルタイム版

議事録転送Bot (`U0B305165M1`) が各案件チャンネルに自動投稿する議事録から、
**提案機会・競争リスク・早急対応** を検知し、通知チャンネルに **議事録単位** で送信する。

## チャンネルID（固定）
- `通知先`: `C0AHUC1VDDK` (`#dxm_提案機会_検知くん`)
- `データソース投稿者`: `U0B305165M1` (議事録転送Bot)

## 目的（思想）
- **MTG の中でアップセル機会・提案機会を拾って通知する**
- 提案すれば取れるはずの案件を取りこぼさない
- マネ＋AMが「ここ提案できる」を把握できる体制をつくる

## 検知対象スコープ
- **対象**: 投稿者が `U0B305165M1`（議事録転送Bot）の投稿のみ

---

## 処理フロー（Pythonが担当する部分）

1. **議事録投稿スキャン**: `from:<@U0B305165M1>` で取得
2. **パース**: 投稿テキストから議事録本文を抽出し、▼決定事項 / ▼タスク<顧客> / ▼タスク<ナイル> / ▼BANT / ▼メモ にセクション分解
3. **社外/社内振り分け**: コール名から3段階判定
4. **期日抽出**: A型（YYYY/MM/DD）/ B型（曖昧表記）
5. **AI判定（このスキルが呼ばれる箇所）**: 各項目に絵文字＋サマリ付与＋ノイズ除外
6. **並べ替え**: セクション内を絵文字優先順位順（🆕 > 🚨 > 🏃 > 装飾なし）に並べ替え（Python側）
7. **通知＆スプシ書き込み**

## Phase 5: AI判定（あなたの役割）

各議事録項目に対し、**絵文字判定**と**サマリ生成**を行う。

### 絵文字体系（3種類、1項目1絵文字）

| 絵文字 | カテゴリ | 性質 |
|---|---|---|
| **🆕** | アップセル機会・**攻める価値ある**予算系 | チャンス・攻める |
| **🚨** | 競争・リスク警告 ＋ 顧客課題 | 守る・気をつける |
| **🏃** | 早急 | 急ぐ（TODO感あってもOK） |
| （空文字 ""） | 情報はあるが上記に該当しない通常項目 | 流し読みOK |

### 🆕 を付ける判断基準（重要・厳密に）

「予算情報あり=機械的に🆕」じゃない。**攻める価値ある時だけ🆕**：

| 🆕 付与する | 例 |
|---|---|
| ✅ 大きい予算・新規予算が出る話 | 「月50万〜1000万円規模、高単価案件で再検討」 |
| ✅ 新規領域への拡大シグナル | 「AI検索対策は次フェーズで提案検討」 |
| ✅ 追加提案・新ソリューション・PoC・フェーズ1・来期推進 | 「Instagram動画の内製化サポート希望」 |
| ✅ 内製化・ノウハウ蓄積・再検討意向 | 「内製or外注で運用ノウハウ希望」 |

| 🆕 付けない（装飾なし） | 例 |
|---|---|
| ❌ 予算情報あるが**攻める価値薄い**（金額小さい・高額難） | 「1本5万・月4本（20万）、高額難」 |
| ❌ 既存施策の進行管理 | 「LP制作の進捗共有」 |
| ❌ 事務的TODO | 「請求書送付」「資料整理」 |

### 5カテゴリーとシグナル語彙

| # | カテゴリ | 絵文字 | シグナル語彙の例 |
|---|---|---|---|
| 1 | **アップセル機会** | 🆕 | 追加提案 / 新規領域 / 新ソリューション / フェーズ1 / 来期推進 / 中長期ロードマップ / PoC / 概念実証 / 拡大 / 新施策 / **次フェーズ / 将来検討 / 次のステップ / 内製化 / ノウハウ蓄積 / 内製or外注 / 再検討意向** |
| 2 | **競争・リスク警告** | 🚨 | コンペ / RFP / 提案依頼 / 複数社提案 / 競合 / 再提案 / 比較検討 / 外部コンサルティング / 体制見直し |
| 3 | **予算系（攻める価値あり）** | 🆕 | 大きい予算（月XX万以上規模） / 新規予算 / 報酬予算 / 概算費用（金額大） |
| 4 | **顧客課題（提案の入口）** | 🚨 | 危機感 / 流入減 / 順位変動 / CRO強化 / リピート獲得 / AIオーバービュー / 検索行動変化 |

### 優先順位（複数該当時のロジック）

1項目に複数カテゴリーが該当した場合、**優先順位 🆕 > 🚨 > 🏃** で**1つだけ**選ぶ：

- 「コンペで新規領域を提案、早急に対応」 → **🆕**（新規領域、最優先）
- 「コンペRFPの再提案」 → **🚨**（コンペ・再提案）
- 「至急、提案資料修正」 → **🏃**（早急のみ）
- 「次回MTGで進捗共有」 → 空（事務的）

### 採用基準（拾い漏れ防止優先・重要）

**「拾える情報があれば拾う」がポリシー**：

| 採用するもの（is_noise=false） | 例 |
|---|---|
| ✅ 数字・金額・ニーズ・時期・状況など**何らかの情報あり** | 「1本5万、高額難」「Instagram動画週1本希望」 |
| ✅ シグナル語彙含まなくても情報あれば拾う（装飾なしで表示） | 「現状の出稿費は月50万」 |
| ✅ 緊急度低くてもニーズ情報あれば | 「SEO強化、AI検索次フェーズ、緊急度低」 |

| 除外するもの（is_noise=true） | 例 |
|---|---|
| ❌ 「未確認」「特になし」「N/A」だけ | 「予算：未確認」「Authority：未確認」 |
| ❌ 完全に空欄・情報なし | 「・」だけ |
| ❌ 進捗確認・社内共有メモのみで判断材料にならない | 「次回MTGで共有予定」 |

▼決定事項・▼タスク（顧客/ナイル）・▼BANT は **情報あれば原則すべて採用**。「特になし」「未確認のみ」だけ除外。

▼メモは「マネ＋AMが議事録元投稿を開かなくても判断に使える情報」のみ採用（雑談・進捗・社内共有メモのみは除外）。

### サマリ生成ガイド（重要・厳密に）

**60字程度（40〜70字許容）の自然な日本語**で、以下を満たす：

- **「何を」「誰が」が分かる**具体的な文章（「いつまでに」は別途【期日】行で表示するのでサマリに含めなくてOK）
- **抽象動詞（対応・検討・確認）は避け、具体的な目的語を含める**
- 元議事録の **会議タイトル・他セクション本文の文脈**も使って、何の話か明示する
- 体言止めOK、句点なくてもOK
- **無理に短縮せず60字使ってもいい**
- ⚠️ **重要：サマリには期日表記を含めない**（「2026/06/16まで」「期日：早急」「2026/06/末」等）。Python 側が別途【期日】行で表示するため、サマリに含めると二重表示になる

#### 良い例 vs 悪い例

| ❌ 悪い（抽象的・文脈欠如） | ✅ 良い（具体的・文脈明示） |
|---|---|
| 7月着手必須、今週中連絡希望 | **動画制作プロジェクト**の7月着手必須、今週中に**マッチ度・スケジュール**連絡希望 |
| AI検索次フェーズで追加提案チャンス | **AI検索対策は次フェーズで提案検討、追加提案の余地あり** |
| 提案機会あり | コンペRFP対応として複数社提案実施、ナイル含む再提案へ |
| ナイル側で対応が必要 | RFP受領後にSEO/コンテンツ施策の再提案を提出 |
| 動画制作、内製化希望 | Instagram動画の内製化・週1本品質確保のサポート希望 |

## 出力スキーマ（Claude API への期待出力）

```json
[
  {
    "item_id": 0,
    "emoji": "🆕",
    "summary": "60字程度の自然な日本語サマリ",
    "is_noise": false
  }
]
```

- `item_id`: 入力で渡された項目の識別子をそのまま返す
- `emoji`: `🆕` / `🚨` / `🏃` / `""`（空文字）のいずれか1つ
- `summary`: 60字程度（40〜70字許容）の自然な日本語サマリ
- `is_noise`: true なら出力から除外

期日抽出と社外/社内振り分けは Python側完結なので、AI出力には含めない。

## 実行時の注意

1. 議事録パースは Python（正規表現＋セクション抽出）で確実に
2. AIには「項目本文」に加えて「**会議タイトル**」「**同議事録の他項目本文**」（meeting_context）を渡すので、それも参照してサマリを書く
3. 絵文字判定は **シグナル語彙の存在チェック→優先順位適用**。1項目1絵文字、複数該当時は 🆕 > 🚨 > 🏃
4. 装飾なし（emoji=""）の項目も**情報があれば通知に含める**（採用基準が緩い、装飾なしでも is_noise=false）
5. ▼メモは「マネ＋AMが判断に使える情報」のみ採用
6. サマリは**60字程度の自然な日本語**で、文脈と具体性を保つ。抽象的な書き方禁止
7. 1回のAPIコールに渡す項目数が多い場合は8項目ずつバッチ処理
